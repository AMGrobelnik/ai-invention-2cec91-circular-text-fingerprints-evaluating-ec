## Full Summary

Selected and prepared the Quora Question Pairs (QQP) dataset from HuggingFace (sentence-transformers/quora-duplicates, pair-class config) for evaluating Circular Text Fingerprints vs MinHash n-gram baseline on near-duplicate short text detection.

Dataset characteristics:
- Source: sentence-transformers/quora-duplicates (pair-class), originally from Quora Kaggle Competition
- Total examples: 404,290 question pairs
- Labels: binary (0 = non-duplicate, 1 = duplicate)
- Class balance: 149,263 duplicates (36.9%) / 255,027 non-duplicates (63.1%)
- Text length: median 9 words per question (short-text domain, ideal for n-gram stress-testing)
- Diverse paraphrase types: lexical substitutions, word-order variation, minor rewording

Schema per example:
- input: JSON string with {"text1": ..., "text2": ...}
- output: binary label as string ("0" or "1")
- metadata_is_duplicate: int (0/1)
- metadata_word_count_1/2: whitespace-split word counts
- metadata_char_count_1/2: character lengths
- metadata_row_index: original dataset row index
- metadata_task_type: "binary_classification"
- metadata_n_classes: 2

Output files:
- full_data_out/ (3 parts, ~55 MB each, 134k-135k examples per part)
- mini_data_out.json (3 examples)
- preview_data_out.json (3 examples, strings truncated)

Validation: schema validated against exp_sel_data_out.json, all required fields present, no nulls.

Why QQP over PAWS: QQP has natural short questions (median 9 words) matching the ideal_dataset_criteria (<40 chars for short pairs, 10-80 word target range). PAWS has longer Wikipedia sentences (avg 17+ words, artificially word-scrambled), less representative of the short-text near-duplicate target domain.
