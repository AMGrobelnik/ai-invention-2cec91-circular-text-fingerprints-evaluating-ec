## Layman Summary

Compares Circular Text Fingerprints against four baselines for duplicate question detection; character 4-gram Jaccard wins (F1=0.656) while CTF (R=3) achieves competitive F1=0.642 on 404k Quora pairs.

## Full Summary

Implemented and evaluated Circular Text Fingerprints (CTF) using iterative ECFP-style neighborhood hashing at radii R∈{1,2,3} against four baselines — MinHash k-shingles (k=1,2,3), character 4-gram Jaccard, word-unigram Jaccard, and TF-IDF cosine similarity — on the full Quora Question Pairs dataset (404,290 pairs, 36.9% duplicates).

Experiment pipeline: loaded QQP from three dependency JSON files (~404k pairs), split 60/40 train/test deterministically by row index, tuned similarity thresholds on train via grid search (granularity 0.01) maximizing F1, then evaluated on held-out test set (~161k pairs). Stratified results by avg word count: ≤6, 7–15, 16+ words.

Key findings (overall F1 on test):
- Character 4-gram Jaccard: F1=0.6557 (best overall)
- CTF R=3: F1=0.6422
- CTF R=2: F1=0.6387
- Word-unigram Jaccard: F1=0.6276
- CTF R=1: F1=0.6260
- MinHash k=2: F1=0.6219
- TF-IDF Cosine: F1=0.5937
- MinHash k=1: F1=0.5840
- MinHash k=3: F1=0.5700

Statistical comparison (CTF R=3 vs char 4-gram Jaccard): McNemar p≈0 (highly significant); bootstrap 95% CI on F1 difference: approx [-0.014, -0.013] (CTF slightly below best baseline). CTF shows its strongest relative performance on short texts (≤6 words), consistent with the hypothesis that iterative neighborhood hashing captures token-context correlations missed by static n-grams.
