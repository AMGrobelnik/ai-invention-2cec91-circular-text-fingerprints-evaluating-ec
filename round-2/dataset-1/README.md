## Full Summary

This dataset artifact prepares two high-quality, human-labeled NLP benchmark datasets for evaluating near-duplicate short text detection:

1. **GLUE QQP Validation** (40,430 examples): Quora Question Pairs from the GLUE benchmark. Each example is a JSON object with question1 and question2 fields, labeled as 'duplicate' or 'not_duplicate'. Questions range from short (≤6 words, 'short' stratum) to medium (7–15 words) and long (16+ words), providing rich stratification for length-sensitive evaluation. Sourced from the official Kaggle Quora competition dataset, widely used in NLP benchmarking since 2017.

2. **PAWS-Wiki Test** (8,000 examples): Paraphrase Adversaries from Word Scrambling (PAWS), Wikipedia subset test split. Each example contains sentence1/sentence2 pairs with high lexical overlap constructed to be adversarially challenging — many non-paraphrase pairs share nearly identical word sets but differ in structure. This cross-domain challenge set (Wikipedia vs. Quora questions) enables robust evaluation of precision/recall tradeoffs. Published at NAACL 2019 (Zhang et al.), 82k HuggingFace downloads.

All 48,430 examples are standardized to the exp_sel_data_out.json schema with:
- `input`: JSON string of text pair (question1/question2 or sentence1/sentence2)
- `output`: 'duplicate' or 'not_duplicate'
- `metadata_stratum`: length category ('short', 'medium', 'long') based on average word count
- `metadata_label_int`: integer label (0=not_duplicate, 1=duplicate)
- `metadata_task_type`: 'binary_classification'
- `metadata_word_count_*`: word counts per text field

Stratification metadata enables ≥100 test pairs per length stratum for statistically meaningful evaluation across short, medium, and long text regimes. Output validated against exp_sel_data_out.json schema.
